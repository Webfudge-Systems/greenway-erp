Getting Started

Sign In

Standard User Login

For Fudge Base and Fudge ERP users.

1
Open Fudge Base or Fudge ERP — you are redirected to /login if not signed in.
2
Enter your email and password.
3
Click Sign in. On success, your session is stored for 7 days.
4
If you belong to multiple organizations, the first organization is selected automatically.
⚠️
Self-registration is disabled. Users must be invited by an organization admin or created during organization provisioning.

Platform Super Admin Login

Super admins log in to Orbit only.

1
Open Orbit — navigate to /login.
2
Enter the platform super admin credentials.
3
On success, you are taken to /organizations.

Non–super-admin users who attempt to log in to Orbit are redirected to /unauthorized.

Session Behavior

Session duration7 days (JWT expiry)
Sign outClears token, org context, and department selection
Expired sessionRedirected to login page
Multi-orgActive organization stored in browser as current-org-id

Organization-Level Roles

Every user belongs to one or more organizations through an organization membership. Each membership has a role.

RoleCodeSummary
AdminadminFull access to all ERP modules

Organization admins can create custom roles in Fudge Base → Roles & Permissions.

Access Control

Permissions & Access Control

How Permissions Work

1
Each user has an organization membership with a role.
2
The role defines a permission matrix for ERP modules.
3
Individual users can have custom permission overrides on top of their role.
4
The API enforces permissions on every request.
5
Frontends hide navigation items and actions the user cannot access.

Default Role Permissions

Admin

  • ERP: Manage on all modules

Manager

  • ERP: Manage on all modules except Settings (Read)

Member

AreaModuleLevel
ERPProjects, Tasks, My Tasks, CalendarWrite
Dashboard, Inbox, AnalyticsRead
SettingsNone

Platform Admin Impersonation

When a platform super admin opens an organization's Accounts or ERP workspace via Orbit:

  • The org ID is set automatically in the browser
  • The admin receives full Admin-level permissions for that organization
  • This allows troubleshooting and setup without a separate user account

3

Troubleshooting & FAQ

I cannot log in
1
Verify you are on the correct app URL — Orbit for super admin; Base or ERP for org users.
2
Check your email and password.
3
Ensure your account is not suspended.
4
For org users, confirm you have an active organization membership.
5
Contact your organization admin if you were never invited.
I see "Unauthorized" after logging in
  • On Orbit: Your account is not a platform super admin. Use Fudge Base or Fudge ERP instead.
  • On Fudge Base Security page: You need the Admin role.
  • On any page: Your role may not have permission for that module. Check with your admin.
I cannot see a sidebar item in Fudge ERP
Sidebar items are hidden based on your role permissions. Ask your organization admin to check Roles & Permissions in Fudge Base.
I cannot create a project
Only Admin and Manager roles can create projects. Members can work on tasks within projects they are assigned to but cannot create new projects.
How do I switch organizations?
If you belong to multiple organizations, your active organization is stored in the browser. Platform super admins switch via Orbit (Open Accounts / Open ERP). Standard users typically work within one organization — contact your admin if you need access to another.
The app shows a network error on login

The API backend may be down or misconfigured. Verify:

  • The API is running (local: http://localhost:1337)
  • NEXT_PUBLIC_API_URL points to the correct backend
  • CORS is configured for your frontend URL
How do I reset my password?
Contact your organization admin. Self-service password reset may depend on deployment configuration.
How do I add a new user to my organization?
1
Sign in to Fudge Base.
2
Go to Users.
3
Click Invite user.
4
Enter email, select role and departments.
5
The user receives an invitation and can sign in once confirmed.
How do I provision a new customer organization?
1
Sign in to Orbit as platform super admin.
2
Go to Create organization.
3
Fill in organization name and owner email.
4
Set status on the detail page.
5
Use Open Accounts to configure the tenant.

Glossary

Key terms used throughout the Greenways Suite — built by Webfudge Systems.

Orbit
Product name for the Organization Manager app
Fudge Base
Product name for the Accounts app
Fudge ERP
Product name for the ERP (enterprise resource planning) app
Platform Super Admin
Highest-level user; manages all tenants via Orbit
Organization (Tenant)
A customer workspace with its own users, data, and settings
Org Membership
Link between a user and an organization, carrying a role
RBAC
Role-Based Access Control — permissions defined per role per module
Module
A functional area (e.g., Projects, Client accounts) with its own permission level
Department
Organizational unit used to scope ERP data
Client Account
A record representing a customer company in Fudge ERP
Slug
URL-friendly unique identifier for an organization or project
JWT
JSON Web Token used for session authentication (7-day expiry)
Strapi
The headless CMS / API framework powering the backend
PWA
Progressive Web App — installable web application with offline support
